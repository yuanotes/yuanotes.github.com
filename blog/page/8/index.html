用snippy代替scrapbook

Apr 25th, 2010

firefox下有些很好用的插件，scrapbook就是其中之一，这个插件可以帮助用户抓取网页上的元素，存储在本地，供用户以后进行查看或者编辑。

chrome出来以后，插件不如firefox是个最大的缺陷，不过我在网上发现一个chrome的插件，snippy，和scrapbook的功能差不多，比scrapbook出色的是，这个插件可以将用户抓取的网页保存到SnipBin或者 Google Docs 中，缺点是我还没有发现保存到本地的选项，不过保存到Google Docs 也是个不错的选择，SnipBin就有点不靠谱了。

坚持做完这件事

Apr 24th, 2010

最近两天我居然没有背单词，而且我还有很多很多事情没有做。

接下来的五一可能可以把这些乱七八糟的事情做完，但是五一答应了老爸老妈回家一趟，所以这些乱七八糟的东西真的很乱七八糟。

可能是因为事情太多，又不顺心压力很大吧。我确实感觉到了那种令人窒息的无形又让人很无奈的压力。

我觉得我不是太忙了，而是感觉很忙，却不知道该怎么做。什么都不做带来的后果是我不想接受的，这是我压力的主要来源。

我决定真的忙起来，已经开始了做的事情就做到底，GRE单词还是接着背，网页还是接着做，学习上的事情该怎么做还是怎么做，不给自己找新的事情做。

可能以后的我会觉得这么做很傻，但是现在的我还是傻一点好，傻人有傻运气。

最近都在用vimwiki记录日记和笔记，感觉功能比不上emacs的muse，但我没办法放弃vim，所以还是继续用下去，过了这一阵压抑的时间再谈其他的吧。

时间真的很不够用啊，现在最缺的资源是时间，最需要的就是坚持。

试用vimwiki

Apr 23rd, 2010

试了很多个人知识管理工具，发现wiki的形式是最好的，这个是没得说的。还有几个条件是必须的：

  1. 在线的平台是比较好的，因为我需要在很多地方、不同的操作系统登陆wiki。
  2. 有本地的平台是比较方便的，这样当我不能上网或者上网受到限制的时候我还是能够编辑wiki。
  3. 免费的。收费的不够钱买，破解的用着不舒服。
  4. 不大也不小。像wikimedia那种太大了，个人用起来比较费劲；像tiddlywiki太小了，单文件也不够靠谱。
  5. 编辑比较方便。
发现了几款软件符合我的要求:
  1. 首先是moinmoin，本地编辑十分好用，但是发现部署在线版本的代价比较大，因为其代码是python。python的空间对我来说，还是贵了一点点。
  2. 其次是dokuwiki，本地搭建比较方便，在线的也很实惠。不过发现本地和在线的进行同步是个问题，还有就是搭建php平台也是挺费力的。
  3. 然后是emacs的muse。这个，没办法，其实学习emacs的使用也是有曲线的……而且我喜欢vim的速度。
  4. 最后就是vimwiki了，优点是我会用vim，编辑文本太舒服了，速度快，文本格式的wiki文件和html格式的文件无论是在线还是本地都不是问题；缺点就是对于多媒体不能很好的支持，貌似还不支持文字下滑线的样式，不过对我来说，其缺点可以无视掉。

人的精力有限

Apr 23rd, 2010

以前看了张颐武的围脖，联想到的。记不起来哪一条了，他的微博实在太多。

做事没什么大小之分，人的精力是有限的。注重于“大”的人就没有精力去处理“小”的事情了，相反，注重于“小”的人没那么多精力处理大事情。一个人为一些小事斤斤计较半天，一看就是最近闲得慌的。

有句话一直不明白，现在能理解了：不要和傻逼争论，结果只能是他把你拉到跟他一个程度，然后把你打败。
