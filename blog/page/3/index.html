Django的blocktrans语法

Sep 13th, 2011

官方文档的blocktrans的的标签在django1.2.5和1.3中不一太一样。

blocktrans的标签中中不支持访问变量名的属性以及list或者dict的成员，也就是user.email这种写法是不允许的。

文档中实例代码是这样的，这种在1.2.5中不支持，但是在1.3中是支持的： 

{% blocktrans with amount=article.price count years=i.length %}
...
{% endblocktrans %}

在1.2.5中必须用老的语法： 

{% blocktrans with article.price as amount count i.length as year %}
..
{% endblocktrans %}

另外，老的语法中,with A as B [and C as D] 中的 and 不能跟count写在一起。可以理解为count和with是两个单独的函数，只能并列放着，我没有看源代码，只是个人理解。新版本的新语法比较简洁，如果没有必要一定要守着1.2.5的话，升到1.3还是不错的。

关于博客

Aug 1st, 2011

很久没有更新博客了。

有时候会想，写博客到底是为了什么？读了很多人的博客，如李笑来的博客，阮一峰的博客，以及刘瑜的博客。他们的博文我几乎是每篇必读。

博客可以传播思想，传播作者的思想。前提是该博客有一定数量的读者。当然，有人喜欢把博客用来当作日记本或者记事本用。

我决定把自己的想法记录下来，这是一个需要坚持的过程。

我想自己写一个博客程序代替wordpress，这次绝对不是一个完美主义者的程序员的幼稚想法。我从事的工作是网络应用开发，这样做可以让工作和兴趣之间取得一个平衡。

我觉得自己每天都在成长，因为我每天都在思考。看了很多的文字、视频，想了很多的事情，却没有记录下来，这是一件比较遗憾的事情。

从今天开始，坚持记录自己的想法，哪怕自己都觉得幼稚。

人很难发现自己的缺陷

Apr 5th, 2011

人，真的很难发现自己的错误，太困难了。

这个社会，没人会当面指出别人的错误，喜欢指责别人的人是最不受欢迎的人。而且，没有人愿意让别人指出错误。

偶尔发现自己有不对的地方，都不太想承认。

如果是别人指出我的错误，那么我会非常愤怒。讽刺的是，我自认为我是一个相当开明的人，我认为我能够接受任何批评与指正，除非那些批评是错误的。

但是，问题来了，我得到了很多人的批评，几乎没有一次我觉得他们说对了。谁说我我就跟谁翻脸，所幸身边的人都能够宽容我。而且，我还经常对我看不惯的人以及事物指手画脚，乱加评论。

我的这些错误，我一直是不知道的。在此之前，我一直觉得，不能接受别人指正的人，都是刚愎自用的白痴。我觉得我是一个能够接受别人批评的正人君子。

我发现这些问题之后，有种惊悚的感觉…

我正在阅读卡内基的书，人性的弱点。当我尝试发现别人的优点的时候，愕然的发现，碰到每一个人，我首先想到的是他或者她的缺点，由此来产生优越感。

我有种惊悚的感觉。我怎么是这么一个人！？！？我不是很有自信吗？我最讨厌的，就是谁谁谁贬低别人，以此来抬高自己。

当我发现我并不是那么受欢迎的时候，我就发现了，问题是出在我身上的。但是我不知道这问题究竟是什么。

首先，我脾气不好，一点小事我就暴跳如雷。我甚至觉得这是男人的本色。说实话，我最讨厌这种人，就像个小丑。

其次，我喜欢抱怨，尤其习惯对着身边亲近的人抱怨。说实话，我瞧不起喜欢抱怨的人。

再次，我喜欢指责别人，只要是我不喜欢的我都指责。是的，我最讨厌没事摆着个八婆脸，对别人指手画脚的人。

其实，每个人都觉得自己是独一无二的，每个人都觉得，自己是比较完美的。如果一个人觉得自己一无是处，那么他或她，就不可能有任何自信，就不会有任何生产力，甚至离自杀不远了。而且，人们自己不完美的时候，往往习惯指责别人，来让自己产生完美的错觉。

纵然本性如此，也有可以改善的地方。

首先，听完别人的指责，不要大动肝火。我们可以不接受指责，但是不能轻易剥夺别人这种权利。因为，人，自己发觉自己的错误，实在是太难。有时候，别人指出错误，我们也难以发觉。

其次，不要指责别人。没人愿意接受指责。我们可以要求自己接受指责，却不能要求别人也这么做。

再次，要善于发现别人的优点，忽略他们的缺点。总是感受这个世界的阴暗面，活得该有多阴暗啊。而且，这么做，不利于交际。

再次感谢阅读给我带来的好处，感谢身边朋友对我的宽容和耐心。我真幸运。

Django如何管理静态文件

Mar 28th, 2011

Django的template确实很强大，但是我们不可能把CSS文件也嵌入到template中。

但是CSS又不能简单的放在template目录下让template调用，因为django的目录结构是由urls.py这个文件来指定的，凡是没有指定的路径，外界都不能访问。

所以，最好的办法，是指定一个目录，该目录下所有的文件都能够被匿名用户访问。

虽然django的官方文档很详细，但是，一般人还是很难看懂其中staticfiles app到底是在扯什么东西。

好在django默认建立的project的settings.py文件中有MEDIA_ROOT，MEDIA_URL等选项。去网上搜索了一把，找到了一篇文章（墙外），差不多可以解决这个问题。虽然，貌似没有staticfiles神马的用起来方便。

首先，在settings.py文件中定义（以下代码都是写文章时候手打的，如有错误请留言指正） 

MEDIA_ROOT=os.path.join(os.path.dirname(__file__),'media').replace('\\','/') #如果是*nix系统，不需要replace函数
MEDIA_URL='/media/' #末尾一定要加上/
ADMIN_MEDIA_PREFIX = '/media/admin/' #填啥都可以，就是不能跟MEDIA_URL一样

然后，在urls.py文件中： 

from django.views.static import * 
from django.conf import settings

#…
#以下是urlpatterns中的
(r’^media/(?P.*)$’, ‘django.views.static.serve’, {‘document_root’: settings.MEDIA_ROOT}),

做完这些之后，测试一下 yourdomain.com/media/default.css
仅仅输入 yourdomain.com/media/ 是看不到任何东西的。

相对的生活

Jan 24th, 2011

枯燥和娱乐，是相辅相成的。这个世界，很多东西都是相对的。贫穷和富有是相对的，如果所有人都是穷人，那么所有的人就很富裕。

寒假在家，最先的设想是很愉快的生活。然而，事实确实相反。这个，我早就应该预料到。因为去年暑假我就没有回家，我以为回家之后，生活会变得非常枯燥。

这么充裕的时间，我居然会觉得无聊。这是一件不太正常的事情。

回想以前，在高中的时候，课间休息时，能有一本杂志看，就是最大的快乐。放假的时候，能玩几个小时的游戏，就是最快乐的事情。然而，现在，那时候最快乐的事情，反而是我现在最不想做的事情。

大三暑假我在一个软件公司实习一个月。那个月让我很讨厌工作，真的，每天准时上班，下班要到五点半，来去都要坐公交车，平时上班老板要求很多，这些束缚都让我受不了。然而，不得不说，那段时间，我学到了很多东西。

罗素说，年轻人最忍受不了的，就是枯燥的生活，他们需要刺激。然而，我却发现，刺激，和枯燥是一对的。没有枯燥就没有刺激。如果一天到晚都刺激，那么，那种生活将会变得越来越无趣，从而促使人们寻找更加刺激的东西。

如果要生活变得有趣，那么，最好的办法是去寻找一点无聊却很有用的事情来做，这样，生活才会多姿多彩。
